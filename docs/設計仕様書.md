# 再生回数マイルストーン — 設計仕様書

## 目的

VTuber（主に音乃瀬奏）の歌動画・カバー動画の再生回数が節目に到達する・したことを検知し、  
ツイートやブラウザ通知で把握できるようにする個人向け支援ツール。

---

## 方針

**方針A（クライアント単体）＋方針B（公開モード）の併用。**

### プライベートモード（デフォルト）
- サーバー不要。単一 HTML ファイルで完結。
- ユーザーが取得した YouTube Data API キーをブラウザの localStorage に保存して使用。
- データ（視聴履歴・マーク・通知済み記録）もすべて localStorage に蓄積。
- GitHub Pages で公開・運用。

### 公開モード（`?mode=public`）
- API キー不要。`public_data.json` + `public_config.json` を静的フェッチして表示。
- データは GitHub Actions（30分ごと）が `scripts/fetch_data.py` を実行して自動更新。
- 更新・通知・設定ボタンは非表示。読み取り専用。

## 機能要件

### MUST（実装済み）

| 機能 | 説明 |
|------|------|
| 動画一覧表示 | チャンネルの全アップロード動画を取得・表示（配信中・未公開・再生数非公開を除く） |
| 再生数確認 | 各動画の現在の再生回数をカード表示 |
| マイルストーン予測 | 次のマイルストーンと予測到達日（ETA）を表示。勢い = 直近72時間の増加速度で算出。1分ごとに再描画してETAを常に最新に保つ |
| 達成ハイライト | 48h以内達成はゴールド枠・発光背景、72h以内達成は薄ゴールド枠で強調表示 |
| ブラウザ通知 | 自動更新時に新規達成を検出 → ブラウザプッシュ通知（★マーク済み動画のみ） |
| ソート切替 | 再生数順 / 投稿日順 / 注目（セクション分け）を切替可能 |
| フィルタ | チャンネル軸（全て/自CH/他CH）とマーク軸（★）の2軸を独立して組み合わせ可能 |
| 手動マーク | ★ボタンで動画をマーク、★フィルタで絞り込み |
| ツイートボタン | 各カードからタイトル・URL・再生数/達成情報を含んだツイート画面へ遷移 |
| 動画リンク | カードクリックで YouTube の動画ページを新タブで開く |
| 再生時間表示 | サムネイル右下にオーバーレイ表示（YouTube スタイル） |
| 投稿日・経過日数 | カード下部に「投稿: YYYY/M/D · N日前」を表示 |
| チャンネル切替 | 設定画面でチャンネルハンドルを変更すると別チャンネルでも動作 |
| 自動更新 | タブを開き続けている間、設定した間隔（デフォルト30分）で自動的に再取得 |
| 他チャンネル動画追加 | 設定画面のテキストエリアに YouTube URL をペーストして任意の動画を追跡対象に追加 |
| データ入出力 | JSON エクスポート / インポートで別ブラウザ・別端末へのデータ移行が可能 |

### WANT（対象外）

- 動画 / ライブ配信 / Shorts の分類タブ  
  → 手動マーク機能で代替可能なため、複雑な分類ロジックは実装しない

## 公開モード設計

### ファイル構成

| ファイル | 役割 | 更新方法 |
|----------|------|----------|
| `public_data.json` | 全動画の再生数履歴（824本）| GitHub Actions が自動更新 |
| `public_config.json` | チャンネル設定・マーク・固定動画ID | 手動編集・コミット |

### public_data.json 構造

```json
{
  "lastUpdated": 1746100000000,
  "ownChannelId": "UCWQtYtq9EOB4-I5P-3fh8lA",
  "videos": {
    "[videoId]": {
      "title": string,
      "thumbnail": string,
      "publishedAt": string,
      "duration": number,
      "channelId": string,
      "history": [{ "ts": number, "views": number }]
    }
  }
}
```

### public_config.json 構造

```json
{
  "channelHandle": "OtonoseKanade",
  "ownChannelId": "",
  "thresholdDays": 3,
  "markedIds": [],
  "pinnedVideoIds": []
}
```

### GitHub Actions ワークフロー（`.github/workflows/update-data.yml`）

- トリガー: 30分ごと cron + 手動実行
- 実行内容: `python scripts/fetch_data.py` → `public_data.json` を上書きコミット
- 必要シークレット: `YOUTUBE_API_KEY`（リポジトリ Settings → Secrets and variables → Actions）
- 同一再生数では履歴を追記しないため、差分なしの場合はコミットしない

### 公開モードの初期化フロー（index.html）

```
URLSearchParams.get('mode') === 'public'
  → publicMode = true
  → 設定ボタン（⚙）のみ非表示
  → refreshBtn を自動更新トグル（togglePublicAutoRefresh）に差し替え
  → 「公開」バッジを表示
  → Promise.all([fetch public_data.json, fetch public_config.json])
    → store.videos, ownChannelId, lastUpdated を上書き
    → markedSet, pinnedVideoIds, thresholdDays を config から設定
    → renderMain()
```

### 自動更新・通知ボタンの状態遷移

| 操作 | ⟳ | 🔔 |
|------|---|---|
| 初期状態 | OFF | OFF |
| ⟳ を押す（OFF→ON） | ON | 変化なし |
| ⟳ を押す（ON→OFF） | OFF | OFF（連動） |
| 🔔 を押す（OFF→ON） | ON（連動） | ON |
| 🔔 を押す（ON→OFF） | OFF（連動） | OFF |

- 自動更新OFFと通知ONの組み合わせは存在しない
- 自動更新間隔: 30分（`PUBLIC_REFRESH_MS = 30 * 60 * 1000`）
- 再フェッチ時はキャッシュ回避のため `?t=タイムスタンプ` を付与
- 再フェッチ後に `checkAndNotify()` を呼び出して節目達成を検出

## データモデル（localStorage）

```
store = {
  settings: {
    apiKey: string,          // YouTube API キー
    channelHandle: string,   // チャンネルハンドル（@不要）
    thresholdDays: number,   // 「もうすぐ」判定閾値（日）
    refreshMinutes: number,  // 自動更新間隔（分）
    ownChannelId: string,    // チャンネル取得時に自動保存される YouTube チャンネルID
    channelFilter: string   // チャンネルフィルタ状態 ('all' | 'own' | 'external')
  },
  videos: {
    [videoId]: {
      title: string,
      thumbnail: string,     // medium サムネイル URL
      publishedAt: string,   // ISO 8601
      duration: number,      // 秒数
      channelId: string,     // YouTube チャンネルID（isExternal 判定に使用）
      history: [             // 最大7日分
        { ts: number, views: number }
      ]
    }
  },
  lastUpdated: number,       // 最終取得タイムスタンプ
  markedIds: string[],       // 手動マーク済み動画 ID 一覧
  pinnedVideoIds: string[],  // 手動追加した動画 ID 一覧（他チャンネル含む）
  notified: {                // 通知済み記録（7日で自動削除）
    "[videoId]:[milestone]": number  // タイムスタンプ
  }
}
```

## 予測ロジック

1. 直近72時間の history を使用。2点未満の場合は全履歴にフォールバック
2. ウィンドウ内の全点に対して**最小二乗法**で増加速度 [views/day] を推定
3. 予測残日数 = (次のマイルストーン − 最終取得時再生数) / 速度 − 最終取得からの経過日数
4. 速度が算出できない場合は「勢い計算中...」と表示
5. 予測残日数が 0 以下の場合は「まもなく」と表示
6. 1分ごとに `renderMain()` を再実行して ETA 表示を常に最新化

> 最終取得からの経過時間を差し引くことで、API 更新なしでも ETA が過去にならない。  
> 両端2点のみを使う線形補間と異なり、中間点が増えるほど推定が安定する。

## 他チャンネル動画の追跡

`store.pinnedVideoIds` に登録した動画 ID は、`fetchAllVideos()` 実行ごとにチャンネル外動画として個別に再取得される。

```
判定: video.channelId !== store.settings.ownChannelId
→ isExternal = true → カードに「他チャンネル」バッジ表示
```

- チャンネル動画と同じ履歴管理・マイルストーン予測が適用される
- 削除時は `store.videos` からも除去（チャンネル動画は次回更新で復活するため除去しない）

## YouTube API 使用量

### プライベートモード（ブラウザ直接実行・1回あたり）

| 呼び出し | クォータ単位 |
|----------|-------------|
| channels.list × 1 | 1 |
| playlistItems.list × N ページ | N（通常 1〜5） |
| videos.list × M バッチ（50本単位）| M（通常 1〜20） |
| videos.list × P バッチ（ピン留め動画）| P（通常 1〜2） |
| **合計目安** | **約 5〜30 単位** |

### 公開モード（GitHub Actions・1回あたり）

| 呼び出し | クォータ単位 |
|----------|-------------|
| channels.list（ハンドル解決、初回のみ）| 1 |
| playlistItems.list × N ページ | N（通常 17〜18） |
| videos.list × M バッチ（50本単位）| M（通常 17〜18） |
| **合計目安** | **約 35〜40 単位** |

30分ごとの自動更新 × 48回/日 ≒ **1,800〜2,000 単位/日**。日次クォータ上限 10,000 単位に対して余裕あり。

## 制約・既知の仕様

- **予測の精度は使用時間に比例する**  
  初回起動直後は history が1点のみのため予測不可。数回の更新後から機能する。

- **マルチデバイス同期は JSON エクスポート/インポートで対応**  
  localStorage はブラウザ・端末ごとに独立。JSON 出力 → 別端末で JSON 取り込みにより移行可能。リアルタイム同期は非対応。

- **ブラウザ通知はページを開いている間のみ有効**  
  タブを閉じた状態では通知は届かない。

- **メンバー限定動画・非公開動画は除外**  
  `statistics.viewCount` が取得できない動画は一覧に含まれない。

- **配信中・公開前の動画は除外**  
  `liveBroadcastContent === 'live' || 'upcoming'` の動画は再生数未確定のため対象外。
