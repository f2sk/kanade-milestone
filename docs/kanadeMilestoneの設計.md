# 目的
- 音乃瀬奏の主に歌動画（カバー、オリジナル曲など）の再生回数が節目に到達する・したことを検知し、認識する助けとする
- 現在の再生回数と次の節目再生回数、予想到達日を知ること
- その動画が規定再生回数に達したことをツイートしたい

# 設計要件

## MUST要件
- 集計対象のデフォルトは音乃瀬奏（ https://www.youtube.com/@OtonoseKanade/ ）とするが、他の配信者でも動作可能な設計とすること
- Github Pagesで公開して運用
- PC, スマホで閲覧可能で、初めて開いてもMUST要件にある動作が可能であること

- 節目の再生回数を事前に検知・予想し、ハイライト表示する
- 節目の再生回数に達したことを検知し、より強調してハイライト表示、ブラウザから通知をする
- 最新順と再生回数順を切替可能
- 各動画のカードにリンクを貼り、その動画のページに遷移可能
- 動画の再生時間をカードに表示する
- 追跡対象の動画を手動でマークし、それをフィルターする機能。マークした動画は記憶し次回以降にも反映させること。
- 各動画のカードには共有リンクを付け、節目の再生回数をツイート内容に含めたツイート画面へ遷移させる機能


## MUSTだが方向性を選択可能としたい要件
### 方針A
- ローカルでの蓄積不要で、単発でページを開いても動作する。
- YouTubeが公式的に認めた手段を取ること。例えばYouTube公式のAPIを使用することはOK。オンライン完結動作の場合はユーザー個人のAPIキーを入力して使用

### 方針B
- Raspberry pi等で常駐し動作した結果をWebに反映し、クライアント側では単発で開いて動作すること。
- YouTubeが公式的に認めた手段を取ること。例えばYouTube公式のAPIを使用することはOK。ラズパイ等のサーバー運用の場合はサーバー側で固定のAPIキーで運用とする。

## WANT要件
- 動画、ライブ配信、Shortsを分類可能
